# 🗡️ Pocket Souls — ASCII Roguelike Engine

A fully procedural, ASCII-based RPG inspired by Dark Souls mechanics, developed as part of a CSU Senior Software Design project.

## Overview

Pocket Souls is a room-based exploration game where players navigate a dynamically generated world, interact with characters, and engage in combat, trading, and resource gathering.

Built from the ground up, this project functions not just as a game, but as a **modular RPG engine** written in C#.

## Key Features

### 🌍 Procedural World Generation

* Fully generated map system using interconnected rooms
* Each playthrough offers a unique layout and experience

### ⚔️ Item & Equipment System

* Abstract item architecture (base → weapon → specific classes like swords)
* Fully generated items with dynamic properties
* Inventory system with weight, capacity, and stats
* Full Armor and weapons system

### 🧠 AI & Game State

* AI-driven characters and enemies
* Internal player state tracking and updates
* Multiple character types with different behaviors

### 💬 Interaction System

* Dialogue/speech system for interacting with NPCs
* Dynamic responses based on context

### 💰 Economy & Gameplay Loop

* Buy, sell, and trade items
* Mine resources in specific rooms
* Fight enemies and bosses in RPG-style combat

### 🖥️ ASCII Interface

* Clean console-based GUI for:

  * Inventory management
  * Trading
  * Combat interactions
  * Navigation

## Architecture

* Object-oriented design with layered abstractions
* Modular systems for:

  * Items
  * Map generation
  * Dialogue
  * Combat
* Designed to be extendable as a reusable engine

## Development Notes

* ~100 hours of development over 2 weeks
* Built with performance and flexibility in mind
* Focused on creating reusable systems, not just a single game

## Future Improvements

* Expanded AI behaviors
* More item classes and rarity systems
* Enhanced dialogue branching
* Save/load system
